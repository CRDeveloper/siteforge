# SiteForge Dashboard Deployment Guide

Comprehensive guide for deploying the SiteForge admin dashboard to ADMINDev infrastructure at `siteforge.YOURADMINDOMAIN.com`.

---

## Overview

The dashboard is a static HTML file generated by the siteforge CLI and served through ADMINDev's Next.js application. No database, no backend processing—just fast, reliable access to site status and metadata.

---

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    User Browser                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
        https://siteforge.YOURADMINDOMAIN.com
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              CloudFront Distribution                        │
│        (Cache + DDoS protection + HTTPS)                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│             Route53 DNS Resolution                          │
│        siteforge.YOURADMINDOMAIN.com → CloudFront           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│            ADMINDev Next.js Application                     │
│  GET /siteforge-dashboard → Serves static HTML file         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│      SiteForge Dashboard HTML (Static, ~50KB)               │
│    - Site cards with metadata                               │
│    - Health status (optional)                               │
│    - Inline CSS (no external resources)                     │
└─────────────────────────────────────────────────────────────┘
```

- AWS account with permissions to manage Route53, CloudFront, ACM

- `dig` or `nslookup` for DNS testing
- `curl` for HTTPS verification

### 2.2 Production Implementation

For production, load the static file at build time or runtime:

**Option A: Build-time loading**
```typescript

```

**Option B: Runtime loading (recommended for updates)**
```typescript
import { promises as fs } from 'fs';
import path from 'path';

async function getDashboardContent() {
  try {
    const dashboardPath = path.join(
      process.cwd(),
      '../../siteforge/apps/admin/siteforge-dashboard.html'
    );
    return await fs.readFile(dashboardPath, 'utf-8');
  } catch (error) {
    console.error('Failed to load dashboard:', error);
    return '<p>Dashboard unavailable</p>';
  }
}

export default async function SiteForgedDashboard() {
  const html = await getDashboardContent();

      srcDoc={html}

      title="SiteForge Dashboard"
    />
  );
}
```

## Step 3: Configure DNS

### 3.1 Get CloudFront Distribution Domain

From ADMINDev CDK deployment:

```bash
cd /workspace/ADMINDev/aws

# List CDK stacks
cdk list

# Get CloudFront output
aws cloudformation describe-stacks \
  --stack-name ADMINDev-CloudFront \
  --query 'Stacks[0].Outputs[?OutputKey==`DistributionDomain`]'

# Example output:
# "d111111abcdef8.cloudfront.net"
```

### 3.2 Create Route53 Record

**AWS CLI Method**:

```bash
# Get hosted zone ID for YOURADMINDOMAIN.com
HOSTED_ZONE_ID=$(aws route53 list-hosted-zones-by-name \
  --dns-name YOURADMINDOMAIN.com \
  --query 'HostedZones[0].Id' \
  --output text)

# Remove leading /hostedzone/ prefix
ZONE_ID=$(echo $HOSTED_ZONE_ID | cut -d'/' -f3)

# Create CNAME record for siteforge subdomain
aws route53 change-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --change-batch "{
    \"Changes\": [
      {
        \"Action\": \"CREATE\",
        \"ResourceRecordSet\": {
          \"Name\": \"siteforge.YOURADMINDOMAIN.com\",
          \"Type\": \"CNAME\",
          \"TTL\": 300,
          \"ResourceRecords\": [
            {
              \"Value\": \"d111111abcdef8.cloudfront.net\"
            }
          ]
        }
      }
    ]
  }"
```

**AWS Console Method**:
1. Go to Route53 → Hosted Zones
2. Click on `YOURADMINDOMAIN.com`
3. Click "Create record"
4. Name: `siteforge`
5. Type: `CNAME`
6. Value: `d111111abcdef8.cloudfront.net` (from CloudFront)
7. TTL: `300`
8. Click "Create records"

### 3.3 Update CloudFront Alternate Domain Names

Update ADMINDev CloudFront distribution to accept `siteforge.YOURADMINDOMAIN.com`:

**CDK Update** (Optional, if not already configured):
```python
# In ADMINDev CloudFront stack
distribution = cloudfront.Distribution(
    # ... existing config ...
    domain_names=['YOURADMINDOMAIN.com', 'www.YOURADMINDOMAIN.com', 'siteforge.YOURADMINDOMAIN.com'],
    certificate=certificate,

```

Then redeploy:
```bash
cd /workspace/ADMINDev/aws
cdk deploy ADMINDev-CloudFront
```

**Checklist**:
- [ ] Route53 record created for `siteforge.YOURADMINDOMAIN.com`
- [ ] Points to CloudFront distribution
- [ ] CloudFront accepts `siteforge.YOURADMINDOMAIN.com` as alternate domain

## Step 4: SSL/TLS Certificate

### 4.1 Verify ACM Certificate

The ACM certificate should automatically include the new subdomain if using a wildcard or if already provisioned:

```bash
# List ACM certificates
aws acm list-certificates \
  --query 'CertificateSummaryList[?DomainName==`*.YOURADMINDOMAIN.com`]'

# Describe certificate
aws acm describe-certificate \
  --certificate-arn arn:aws:acm:us-east-1:123456789012:certificate/12345678-1234-1234-1234-123456789012 \
  --query 'Certificate.SubjectAlternativeNames'
```

### 4.2 If Certificate Needs Update

If `siteforge.YOURADMINDOMAIN.com` is not in the certificate:

1. Request new certificate in ACM for `siteforge.YOURADMINDOMAIN.com`
2. Validate via DNS (add CNAME to Route53)
3. Wait for certificate to be issued (~5-10 min)
4. Update CloudFront to use new certificate
5. Redeploy: `cdk deploy ADMINDev-CloudFront`

**Checklist**:
- [ ] SSL certificate valid for `siteforge.YOURADMINDOMAIN.com`
- [ ] Certificate not expired
- [ ] CloudFront using correct certificate

## Step 6: Set Up Automated Dashboard Generation

### 6.1 Create GitHub Actions Workflow

**File**: `.github/workflows/generate-dashboard.yml`

```yaml
name: Generate SiteForge Dashboard

on:
  push:
    branches: [main]
    paths:
      - 'siteforge/sites/**/site-config.json'
      - 'siteforge/cli/siteforge/commands/dashboard.py'
  workflow_dispatch:
  schedule:
    - cron: '0 0 * * *'  # Daily at midnight UTC

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'

      - name: Install dependencies
        run: |
          pip install -r /workspace/siteforge/requirements.txt

      - name: Generate dashboard
        run: |
          cd /workspace/siteforge
          python -m cli.siteforge dashboard --ping

      - name: Copy to ADMINDev
        run: |
          mkdir -p /workspace/ADMINDev/nextjs-admin-cms/public
          cp /workspace/siteforge/apps/admin/siteforge-dashboard.html \
             /workspace/ADMINDev/nextjs-admin-cms/public/

      - name: Commit changes
        run: |
          cd /workspace/ADMINDev
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add nextjs-admin-cms/public/siteforge-dashboard.html
          git commit -m "chore: update siteforge dashboard" || true
          git push

      - name: Slack notification
        if: always()
        run: |
          # Optional: Send Slack notification of dashboard update
          echo "Dashboard updated at $(date)"
```

### 6.2 Update Main Deployment Workflow

## Troubleshooting

### Dashboard Not Displaying

**Problem**: Page shows blank or loads indefinitely

**Solution**:
1. Check browser console for errors
2. Verify HTML file exists: `ls -l apps/admin/siteforge-dashboard.html`
3. Regenerate dashboard: `python -m cli.siteforge dashboard`
4. Check CloudFront cache invalidation needed

**Action**:
```bash
# Invalidate CloudFront cache
aws cloudfront create-invalidation \
  --distribution-id E123456EXAMPLE \
  --paths "/siteforge-dashboard/*" "/"
```

### DNS Not Resolving

**Problem**: `siteforge.YOURADMINDOMAIN.com` DNS lookup fails

**Solution**:
```bash
# Check DNS record
dig siteforge.YOURADMINDOMAIN.com

# If not resolving, check Route53
aws route53 list-resource-record-sets \
  --hosted-zone-id $ZONE_ID \
  --query "ResourceRecordSets[?Name=='siteforge.YOURADMINDOMAIN.com.']"
```

### SSL Certificate Error

**Problem**: "SSL certificate invalid" warning in browser

**Solution**:
1. Verify CloudFront has correct certificate
2. Check certificate includes `siteforge.YOURADMINDOMAIN.com`
3. Wait for certificate propagation (5-10 min)
4. Hard refresh browser cache

### Site Links Not Working

**Problem**: Clicking site domain opens blank page

**Solution**:
1. Verify site domain is correct in `site-config.json`
2. Check site is actually online
3. Verify HTTPS is enabled on site
4. Check browser sandbox restrictions don't block navigation

### Health Checks Not Updating

**Problem**: Status indicators show "unknown" or wrong status

**Solution**:
1. Regenerate with `--ping` flag: `python -m cli.siteforge dashboard --ping`
2. Verify sites have `/api/config` endpoint
3. Check network connectivity from CI/CD environment
4. Increase timeout if sites are slow: Modify `dashboard.py`

## Performance Tuning

### CloudFront Caching

**Recommendation**: Set aggressive caching for static dashboard

```python
# In ADMINDev CDK CloudFront stack
cache_policy = cloudfront.CachePolicy(
    self, 'DashboardCachePolicy',
    cache_compress=True,
    default_ttl=cdk.Duration.hours(1),
    max_ttl=cdk.Duration.days(1),
    min_ttl=cdk.Duration.minutes(0),
    cookie_behavior=cloudfront.CacheCookieBehavior.none(),
    header_behavior=cloudfront.CacheHeaderBehavior.none(),
    query_string_behavior=cloudfront.CacheQueryStringBehavior.none(),
)
```

### Page Load Optimization

- ✅ Inline CSS (no external requests)
- ✅ No JavaScript (instant rendering)
- ✅ Minimal HTML (~50KB)
- ✅ CDN caching via CloudFront
- ✅ GZIP compression (automatic via CloudFront)

**Expected Performance**:
- First byte: <100ms (via CDN)
- Full page load: <1s
- Mobile: <2s (on 4G)
